## Visual Aid Style Guide

This style guide defines how to design, adapt, and use visual aids in
the workbook:

**"Teaching for Impact: Designing Effective & Open Training"**

Visual aids must support learning, decision-making, and real-world
application---not decoration.

------------------------------------------------------------------------

## 1. Design Principles

### When to use visuals

Use visuals when they:

-   explain processes or workflows\
-   show relationships between concepts\
-   support decisions\
-   simplify complexity\
-   improve recall

### High-value visuals

A high-value visual:

-   communicates one core idea\
-   reduces text\
-   is understandable at a glance\
-   supports application\
-   aligns with a specific learning objective

### What to avoid

-   decorative visuals\
-   overcrowded diagrams\
-   inconsistent styles\
-   unnecessary colour variation\
-   text-heavy visuals

------------------------------------------------------------------------

## 2. Colour System

### Talarify palette

-   #1d647d --- primary teal\
-   #638fa8 --- muted blue\
-   #9cb5cb --- light blue-grey\
-   #507d8e --- teal-grey\
-   #d46852 --- accent coral\
-   #004b6c --- dark teal

### Colour roles

-   primary → main structure\
-   secondary → supporting elements\
-   background → grouping\
-   neutral → connectors\
-   accent → decisions / highlights\
-   emphasis → key nodes

### Colour rules

-   use max 3--4 colours per visual\
-   keep colour meaning consistent\
-   use accent sparingly\
-   ensure contrast for readability

------------------------------------------------------------------------

## 3. Typography

### Fonts

-   Inter\
-   Source Sans Pro\
-   Arial

### Typography rules

-   use short phrases only\
-   avoid sentences\
-   maintain strong contrast\
-   keep hierarchy simple (title, labels, supporting text)

------------------------------------------------------------------------

## 4. Layout System

### Flow layout

Use for:

-   processes\
-   sequences\
-   workflows

Rules:

-   left-to-right direction\
-   evenly spaced steps\
-   minimal crossing lines

------------------------------------------------------------------------

### Cycle layout

Use for:

-   feedback loops\
-   iteration\
-   continuous improvement

Rules:

-   clockwise direction\
-   clearly defined stages

------------------------------------------------------------------------

### Triangle layout

Use for:

-   balance between three elements\
-   trade-offs

Rules:

-   equal emphasis unless justified

------------------------------------------------------------------------

### System map

Use for:

-   complex relationships\
-   interacting systems

Rules:

-   group related elements\
-   minimise visual clutter

------------------------------------------------------------------------

### Decision tree

Use for:

-   decision-making\
-   branching logic

Rules:

-   clear questions\
-   clear outcomes\
-   limited depth

------------------------------------------------------------------------

## 5. Shapes and Components

### Standard shapes

-   Rounded rectangle → default\
-   Rectangle → structured steps\
-   Circle → cycles\
-   Diamond → decisions\
-   Container → grouping

### Shape rules

-   use rounded corners consistently\
-   avoid heavy borders\
-   use simple fills\
-   maintain spacing

------------------------------------------------------------------------

## 6. Arrows and Connections

### Arrow rules

-   simple and consistent\
-   medium thickness\
-   clear direction

### Direction

-   left-to-right (default)\
-   clockwise for cycles

### Highlighting

-   neutral arrows for most flows\
-   accent colour for key transitions

------------------------------------------------------------------------

## 7. Icon Usage

### Use icons when:

-   they improve recognition\
-   they support quick scanning

### Rules

-   use a single icon set\
-   keep icons simple\
-   always include labels

### Avoid

-   decorative icons\
-   mixed styles

------------------------------------------------------------------------

## 8. Background and Export

### Background

-   white (default)\
-   light grouping backgrounds optional

### File formats

-   SVG (preferred)\
-   PNG (fallback)

### Naming

module-section-topic-type.svg

### Folder structure

docs/assets/visuals/

## 10. Consistency Rules

-   same colours = same meaning\
-   reuse layout types\
-   align with workbook terminology\
-   maintain spacing and alignment

------------------------------------------------------------------------

## 11. Quality Checklist

-   improves understanding\
-   communicates one idea\
-   readable at normal size\
-   minimal text\
-   consistent design\
-   reusable\
-   correctly attributed

------------------------------------------------------------------------

## 12. Visual Priorities for This Workbook

Prioritise these visuals:

-   training design workflow\
-   system map of training context\
-   learning cycle\
-   reuse/adapt decision tree\
-   licensing decision visual\
-   sharing and sustainability flow

------------------------------------------------------------------------

## 13. Summary Rules

-   one visual = one idea\
-   default layout = flow\
-   default shape = rounded rectangle\
-   default colour = primary teal\
-   use accent only for emphasis\
-   keep text minimal\
-   prefer SVG\
-   ensure reusability
